Create Fraud Check

Main check module. Use this to screen suspicious websites for phishing-style indicators before users click.

  1. Paste the full URL including protocol (`https://` or `http://`).
  2. Run check to score URL structure and risk signals.
  3. Review reasons in feed and decide block / monitor policy.

Secondary module. Upload files to triage potentially malicious content before sharing internally.

  1. Select one file to upload for heuristic analysis.
  2. Run check to inspect extension, entropy, and script markers.
  3. Use feed output to escalate high-risk files quickly.

Third module. Test payment events for transactional fraud behavior patterns.

  1. Enter transaction amount, channel, card presence, and hour.
  2. Run check to apply model-based probability scoring.
  3. Use risk level and reason codes for analyst triage actions.
